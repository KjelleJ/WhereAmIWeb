My requirements for migrating the whereami app from Android to web include:
1. A responsive web design that can adapt to different screen sizes and devices. Focus on mobile portrait.
2. Use plain JavaScript
3. Avoid CSS. Use Bootstrap for styling and layout.
4. Keep the look and feel of the Android UI as much as possible.
5. Implement the same functionality and features as the Android app.
6. Use the same color scheme and branding as the Android app.
7. Use OpenStreetMap for the maps.
8. Use the same APIs if possible, or find suitable alternatives for any Android-specific APIs.
9. Put the code for the Web app in the Web folder of the project.
10. The Android folder contains the code for the Android app. README.md describes the Android app.
11. Create a new README.md file in the Web folder that describes the Web app and how to run it.
12. Don't use any graph libraries for the Web app, as the Android app does not use any. Instead, implement any necessary graphs using plain JavaScript and HTML5 canvas if needed.
14. Ensure that the Web app is accessible and works well on all modern browsers.
15. The migration should be done step by step so I can review and test each part of the Web app as it is developed. This will help ensure that the final product meets all requirements and functions correctly.
16. Provide clear documentation and comments in the code to explain the functionality and structure of the Web app.
17. If possible make the app runnable after each step of the migration process, so I can see the progress and test the functionality as it is developed.
